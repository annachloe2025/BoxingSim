# CLAUDE.md — ボクシングシミュレーター

## プロジェクト概要
フレーム単位でボクシングの攻防を再現する、戦闘エンジン型のシミュレーター。
格闘ゲームのフレームデータ（始動・発生・硬直）の考え方をボクシングに応用する。
能力値をサイコロで振って勝敗を出すタイプではなく、間合いとタイミングの噛み合いから
自然に挙動が生まれることを目指す。

## 設計方針
- 距離: まず1次元（前進・後退・間合い）で実装。将来的に2次元へ拡張する。
- 行動の判断: 人間の手入力に加え、確率型ユーティリティAI（性格パラメータ）を実装済み。AI対AIで自動対戦可能。
- 被弾の影響は段階的に導入中。
  - 導入済み: HP・ダメージ・ガードによる防御・KO（勝敗）。最適化のゴール（勝敗/残HP等）を測れるようにするため。
  - 未導入: のけぞり（被弾で動作中断）・カウンター割増・ダウン/立ち上がり・スタミナ・反応ラグ（知覚の曖昧化）。
- 技術: ブラウザ（HTML / JS、単一ファイル）。表示は文字ベースで十分。

## コアモデル
- 時間はフレームで刻む。
- キャラクターは Player と 敵 の2体。1次元の座標を持つ。
- 各行動（技）は「始動フレーム → 発生フレーム（当たり判定アクティブ）→ 硬直フレーム」の
  順に進行し、その間は他の行動を取れない（動けないタイミングがある）。
- 状態は「待機 / 始動 / 発生 / 硬直」のいずれか。
- 攻撃にはリーチを設定する。発生フレーム中に相手が間合い内にいれば「接触」とみなす。
  現段階では被弾効果を与えず、接触イベントとして記録するのみ。
  接触時の相手の状態（待機/始動/発生/硬直）も記録し、刺さった・お見合い・相打ちを観察可能にする。

## フェーズ1（現在）
モーション作成ツールのように、フレームを再生ボタンで進められる最小システム。
Player と敵がいて、各キャラの状態を常に文字で確認できる。

## 開発ルール
- 作業を始める前に必ず相談する。仕様を勝手に決めて作り込まない。
- 変更内容は LOG.md に日付で記録する。
- 今後のタスクは TASKS.md で管理する。
