フェーズ1: 最小システムの完成

日付: 2026-05-24 ／ 場所: BoxingSim / index.html
プロジェクト開始日。方針合意から、フレームを再生ボタンで進められる最小システム（再生・コマ送り・コマ戻し・接触イベントログ）までを1日で構築した記録。 被弾効果は未導入、接触をログに記録するのみ。

1. プロジェクト方針合意

初日の相談で確定した5本の柱。

  • フレーム単位の戦闘エンジン型シミュレーター（格ゲーのフレームデータをボクシングに応用）
  • 距離は 1次元から開始、将来的に2次元へ
  • 行動の判断は当面手動入力、将来的にAI化
  • 被弾の影響（ダメージ・勝敗等）は当面扱わない。フレームの噛み合いの観察を優先
  • 技術はブラウザ（HTML/JS）、表示は文字ベース

この5本のうち「文字ベース・1次元・被弾未導入」の3つは「機械的仕組みを先に積む」設計理念から導出されている。CLAUDE.md / LOG.md / TASKS.md の3点を同日に作成。

2. 最小フレームエンジン（index.html）

モーション作成ツールのように、フレームを1コマずつ動かせる土台を作る。

操作意味
再生 / 一時停止連続フレーム進行のオン・オフ
コマ送り1F だけ進める
コマ戻し1F だけ戻す（履歴を保持）
リセット初期状態に戻す

キャラクターは Player と 敵 の2体。各キャラに 1次元座標と、4種の状態 待機 / 始動 / 発生 / 硬直 を持たせ、毎フレーム常時表示する。

3. 技データと行動列の編集

各技は 始動 → 発生 → 硬直 の3区間を持つ。表で値を編集でき、行動列（フレーム毎に何を出すか）も入力可能。

この時点では能動的に「振る」だけの存在で、ガード・キャンセル・AI などは未実装。

4. 接触判定と接触イベントログ

攻撃にはリーチを設定。発生フレーム中に相手が間合い内にいれば「接触」とみなす。

  • 現段階では被弾効果を与えず、接触イベントとして記録するのみ
  • 接触時の相手の状態（待機 / 始動 / 発生 / 硬直）も記録
  • これにより「刺さった / お見合い / 相打ち」を観察できる

5. UI 調整

  • プレビューに収まるよう全体をコンパクト化（余白・文字・ログ高さを縮小）
  • キャラカードに状態と同じ大きさで「現在の行動（前進 / ジャブ等）」を併記
  • 間合いゲージ上に攻撃のリーチを帯で可視化: 発生中=濃 / 始動中=淡 / 自分=緑・敵=赤
  • 凡例を追加

「自分の現在状態」「相手の現在状態」「両者の射程帯」を1画面に同時表示できる構成。次フェーズ以降の議論の土台になる。

6. この日の到達点

フレーム単位の進行・状態表示・接触記録という観察のための最小単位が揃った。 被弾効果（ダメージ・のけぞり・KO）は意図的に未導入のまま、まず「フレームが噛み合うのを見る」段階を確保。

翌 2026-05-25 から路線が FSM 設計（ステートマシンとして「カチッと」定義する方向）へ切り替わる。
