技データベース参照層 Step 1a — 実装完了

既にある技データと現況を AI が参照できる surface を作る、挙動を変えない配管のみのステップ。
仕様: docs/design/move_reference_spec.html

関連: move_reference_spec.html（仕様） ／ LOG.md ／ TASKS.md ／ Dashboard

1. 目的

AI が行動を判断する際に「相手の技」と「今の状況」を一望できる参照面を用意する。 これは「相手の技を 5f 知覚した時の状況を技 DB で参照する」仕組みの第一スライス。 既存の MOVES / buildObs / busyFrames を呼ぶだけの薄い層で、挙動は一切変えない（既存コードは未編集、追加のみ）。 scoreActions には未接続。

2. 追加した参照 API

src/index.html の buildObs 直後（1283–1311 行付近）に挿入。

関数戻り値
allMoveKeys() MOVE_ORDER を返す薄い getter。技キーの一覧。
moveData(key) 1 技の素のフレームデータを構造化: key / kind / target / startup / active / recovery / reach / damage / hitstun / blockstun / chipRatio / staminaCost。 未知 key は null。スタミナ実効値は混ぜない素の値。
situation(obs) 既存 buildObs の戻り値を self / opponent / between に集約。
self.{state, phaseRemaining, framesUntilActable, guardTarget, attackTarget}
opponent.{同上 + isAttacking, isGuarding}（★detailLag 観測値を中継）
between.{distance, closing, inMyReach, inOppReach, frameAdvantage}

framesUntilActable は既存 busyFrames の別名で、新規計算はしていない。
frameAdvantage（ガードさせた後の有利/不利 F）はユーザー承認のうえ含めた。
