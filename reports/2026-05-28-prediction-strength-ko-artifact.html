この検証が覆したもの。 前回 Step C 検証は「KO が増えた／判定に行かない／chip 0.3 が弱いから次はガード崩し攻撃」と結論づけた。 本検証は同じ verify.js で2点を測り直し、(1)「KO 増加」の大半は ラウンド構造による見かけ（時間切れが KO に化けただけ）であり、 (2) 判定到達率は 予測強度 modelTrust を上げれば単調に上がるレバーであることを示した。 理念整合: 「ガード崩しが面白そうだから入れる」と逆算するのではなく、 機械的観察で前提（KO 実体・予測の効き）を検証し直し、前回の早計を訂正した。

1. 背景 — ユーザーの2つの疑問

段階5 配管が完成し、Step C 検証で「判定は飾り、chip が弱い、次はガード崩し」と結論づけた。これに対しユーザーが2点を疑った。

Q1 — 「KO が増えた」は実体か、見かけか

旧構成は 800F 単発。新構成は 3R × 300F でラウンド境界ごとに AI・距離・フレーム状態がリセットされる。 総フレームが伸び、境界で停滞がリセットされるなら、同じ膠着が「時間切れ」ではなく「KO」に届くだけではないか。 round 構造が新しい殴り合いを生んだのか、それとも決着しきれない試合を KO に化けさせているだけなのか。

Q2 — 判定が増えないのは「予測 OFF」で測ったからではないか

判定システムの本来の動機は「予測（推論）を戦闘で使うと守備が強まり、判定勝負が増える」だった。 ところが事実として —

  • verify.js の予測強度 modelTrust（以下 MT）の既定は MT=0（予測 OFF）。
  • Step C 検証は MT=0.3 でしか測っていない（当初プラン 0.5 から、守備過多を理由に下げた控えめ値）。

推論を強くすれば判定到達率は上がるのか。それを MT スイープで確かめる、というのが Q2。

判定到達率の定義: (判定 P 勝 + 判定 E 勝 + ドロー) / 50 × 100%。KO・時間切れ以外で決着した割合。

2. 検証の衛生（canonical 不可侵）

  • round-off アブレーション用に一時コピー tools/_verify_tmp_roundoff.js を作成 → 使用 → 削除済み。
  • canonical tools/verify.js は MD5 不変・無編集を確認。
  • src/index.html も 無変更。

実装・検証ハーネスのどちらにも手を入れず、引数・環境変数で挙動を振っての観測。前回 Step C と同じ土俵で測り直している。

3. 実験A — 予測強度スイープ（roundEnabled=true / 3R、MT=0→0.9）

各シナリオ 50 戦。MT を 0 / 0.3 / 0.5 / 0.7 / 0.9 と振り、判定到達率（%）を測る。シナリオ名の数字は presenceLag/detailLag。

数値は判定到達率（%）。緑=MT0.9 での最大値。灰=全 MT で 0%。

解釈 — MT に対する判定到達率の2つの帯

  • 単調増加する帯: 両守備型と低 lag 非対称。最顕著は 両守備 3/5（4 → 28%）と非対称 0/0（44 → 64%）。 両守備 3/5 は MT0.7 以上で跳ねる（10 → 22 → 28）。守備寄りで観測が見える局面では、予測を強めると先回りガードが増え、決着がつかず判定に持ち越される。
  • 効かない帯: 高 lag（5/8・7/10）と攻撃絡み。MT0.9 でも判定率ほぼ 0%。観測が空白すぎて予測が拠り所を失い、機能しない（後述 Q2 結論の注意点）。

傍証 — KO 中央フレームも MT 上昇で伸びる

判定に届かなかった試合でも、MT を上げると決着までのフレームが延びる（=守備寄与で試合が長期化）。

非対称 0/0 KO中央F855816870896827

試合長期化は判定到達の単調増加と同じ向き。予測が守備に効いて決着が遅れている傍証。

4. 実験B — roundEnabled false(旧800F単発) vs true(3R)、同一 MT

「KO 増加が実体か見かけか」を直接見るため、ラウンド構造のオン／オフだけを切り替えて同一シナリオ・同一 MT で比較する。

シナリオ / MTmodeKO率判定時間切れ

非対称 3/5 · MT0off50/5000

両守備 3/5 · MT0off32/50018

解釈 — 時間切れ → KO への変換

  • 既に決着していた帯（非対称 3/5）は off / on とも 50/50 KO で不変。round 構造は何も足していない。
  • 膠着帯（両守備 3/5）では off の時間切れが on でほぼ全て KO 側へ移る: MT0 で 時間切れ 18 → 0・KO 32 → 48、MT0.3 で 時間切れ 29 → 0・KO 21 → 46。 判定はわずか（0 → 2、0 → 4）。
  • つまり on で増えた KO の正体は、off では時間切れだった試合の救済（KO への化け）。新しい殴り合いが立ち上がったわけではない。

5. Q1 結論 — KO 増加は「ほぼ見かけ（混在寄り）」

⚠ KO 増加の大半はラウンド構造の副作用

旧 800F 単発で時間切れだった膠着試合が、3R 化でほぼ全て KO 側へ変換（救済）された。 両守備 3/5 · MT0.3 では off の 時間切れ 29 が、on で KO 21 → 46・判定わずか 4 に化けている。

既に決着していた高 lag／攻撃寄りシナリオ（非対称 3/5・5/8）は 両モードとも 50/50 KO で不変。 増えた KO 分は「総フレーム増 + ラウンド境界での停滞・距離リセット」で説明でき、 round 構造が新たな殴り合いを生んだのではなく、決着しきれない膠着を時間切れではなく KO に化けさせている。

したがって前回の「KO が増えた → ガード崩しが必要」という結論は、この見かけ KO 率に引きずられた早計だった。

6. Q2 結論 — 予測強度を上げると判定到達率は上がる（条件付き・単調）

○ ユーザー仮説「推論を強くすると守備が強まり判定勝負が増える」は支持

両守備型と低 lag 非対称で、MT に対し判定到達率が単調に増加した。 最顕著は 両守備 3/5（4 → 28%、MT0.7 以上で跳ねる）と非対称 0/0（44 → 64%）。 KO 中央フレームも MT 上昇で伸び、予測が守備に効いて試合が長期化する傍証と一致する。

⚠ ただし高 lag 帯・攻撃絡みは別の構造問題

高 lag 帯（5/8・7/10）と攻撃絡みのシナリオは MT0.9 でも判定率ほぼ 0%。 予測が観測空白で機能していない（拠り所となる観測がそもそも入ってこない）。 これは予測強度では解けない別軸の構造問題として浮上した。
