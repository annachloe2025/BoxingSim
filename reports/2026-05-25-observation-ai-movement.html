観測スキーマ・ルールAI・連続移動モデル・ユーティリティAI

日付: 2026-05-25 (後半) ／ 場所: BoxingSim / index.html
AI対戦に向けた4つの基盤投入: ①観測（observation）スキーマ ②単純ルールAI ③連続移動＋キャンセル ④ユーティリティAI（性格4軸）。 完全情報・無遅延の「相打ちジャブのループ」が、非対称な性格を入れた瞬間に崩れることまでを確認した日。

1. 観測（observation）スキーマ

AI に渡す情報を3区分に整理。後の「曖昧化（知覚遅延）」のため ★印で曖昧化対象 を明示しておく。

区分内容備考
self完全把握全フィールド真値
opponentphaseRemaining / action は ★ 後で曖昧化本来人間には見えない
relative間合い・接近・届く技・射程内幾何情報
  • 観測は 毎フレーム両者ぶん 生成
  • AI の出力は行動1つ、行動不能中の入力は無視する想定
  • index.html に buildObs 実装と画面表示。完全データ版
  • Node で値の整合を検証済み

2. 単純ルール AI

固定 if 文の3条件で書いた最初の AI。

  1. 相手攻撃中で射程内 → ガード
  2. ジャブ届く → ジャブ
  3. 遠い → 前進

1フレーム内の評価順による不公平（後評価側が即反応）を修正し、両者が同じ局面を見て同時決定 する方式に変更。

完全情報＋ラグなしだと 相打ちジャブの反復ループ に収束 → これが「曖昧さを後で導入する」動機として記録される。

3. 連続移動 + キャンセル モデル

実ボクサーの感覚に寄せて、移動を「フレーム単位の連続入力」に刷新。

移動種速度性質
普通移動（前 / 後）0.4開始1F のみキャンセル不可
ガード移動（前 / 後）0.2常時ガード
ステップイン / ステップバック0.6ガード不可、停止時1F硬直

キャンセル＝状態ごとの受付入力方式

状態受け付ける入力
アイドル全受付
移動中ジャブ / ストレート / ガード
攻撃中受付なし

将来のコンボ設計にも流用可能な汎用形。AI が毎フレーム意図を返す形で 連続移動 を実現。 間合いに入った瞬間にジャブへキャンセルされ、縁付近（一歩の行き過ぎ）で止まることを確認。

観測の canAct は「キャンセル可能か」に意味更新。stateLabel / charCard / 技編集テーブルも新仕様へ。

注: FSM 図・マトリクス・カード（boxer_fsm*.html）はまだ旧移動のまま。後日更新予定。

4. ユーティリティ AI （性格4軸）

固定 if 文ルールから置き換え。各行動を観測＋性格で点数化し、確率的に選ぶ。

性格4軸

  • 攻撃性 / 防御性 / 好む間合い / 我慢強さ
  • UI スライダーで Player・敵ごとに設定

選択式

確率 ∝ scoreSCORE_POW ／ DECIDE_EVERY=3F（毎フレームのジッタ防止）
  • 各行動の点数・選択確率を画面表示（選択を ▶ 表示）
  • 1位以外も選ぶ ことを確認

将来のカウンター型 / 持久型 / パワー型 / ヒットアンドアウェイ型は、この4軸のプリセットとして実装予定。

5. 非対称な性格で鏡写しループが崩れる

攻撃型 P × 守備型 E のような非対称設定で、Node 検証により以下の創発挙動を確認。

  • 踏み込み
  • 間合い取り直し
  • ガード
  • 差し返し

完全情報＋無遅延でも「性格の非対称さ」が局面を非定常にできることが分かった。 一方で同型同性格では依然として相打ちループが残るので、後の知覚曖昧化の動機は維持。
