FSM 設計とドキュメント化

日付: 2026-05-25 (前半) ／ 場所: BoxingSim / boxer_fsm.html ほか
路線をステートマシンの「カチッと」した設計に切り替えた日。状態11個＋遷移23本を、mermaid俯瞰・遷移マトリクス・状態カードの3表現で並べ、整合を取った。 分類軸として「遷移きっかけ4分類」「4グループ構成」も別 HTML にまとめた。

1. 路線変更: 「カチッと」した FSM へ

フェーズ1の最小エンジンに対し、1人のボクサーの状態遷移を 形式的に定義 していく方向へ切替。 まず1人ぶんを完全に固めてから2人の絡みに展開する設計順序。

2. boxer_fsm（初版たたき台）

  • boxer_fsm.mermaid として 状態11個＋遷移 を定義
  • boxer_fsm.html でブラウザに mermaid 描画
  • 文字色が mermaid 内部スタイルに負ける問題 → 描画後に JS でインライン !important 指定する方式で解決
  • 拡大・縮小・移動を実装（ホイール / ドラッグ / ボタン）

3. 3通り併記による視認性向上

同じ FSM を3つの粒度で並べ、目的別に読めるようにした。

表現役割
(1) mermaid 図俯瞰。全体の遷移網を見る
(2) 遷移マトリクス表状態 × きっかけ の一覧
(3) 状態カード1状態ずつの詳細定義

3者の 遷移定義（全23本）が一致することを確認 済み。

4. 未決定として記録した3点

  • ※1 移動キャンセルの可否
  • ※2 硬直キャンセルの可否
  • ※3 ガードの型（押し続け / 時間）と被弾系の保持フレーム

この3点はその後の連続移動モデル化・ガード統合・のけぞり実装で順次解消されていく。

5. boxer_states_taxonomy.html — 遷移きっかけの4分類

純 HTML / CSS で作成。Claude 側の意見4点も併記。

① 自発

本人が選んで遷移する。アイドルからの攻撃選択・ガード構えなど。

② 自動（時間）

フレーム経過で勝手に進む。始動→発生→硬直→アイドルの自然進行。

③ 外発（相手）

相手側の事象で遷移する。被攻撃・ガード成立など。

④ 強制（審判 / ルール）

ルール側で遷移を強制する。ダウン・試合終了など。

補足: 「被攻撃」は1イベント＋文脈（受けた瞬間の自分の状態）で結果が分岐するという考え方を提示。 これが後の「カウンター分類（始動 / 発生 / 硬直）」の下敷きになる。

6. boxer_fsm_groups.html — 4グループ構成

状態を機能で4つに括り、mermaid 図に再配置。

グループ含む状態
① ベースアイドル / 移動 など
② 自発行動攻撃の始動・発生・硬直、ガード姿勢
③ 被弾リアクションのけぞり、ガード硬直など
④ 強制ダウン・KO・試合終了
