ボクシングシミュレーターを作ること自体は確かに目的の一つだが、本質はそれだけではない。

このプロジェクトの本当の目的:
- ステートマシン(FSM)を作って「リアル」(=人間がボクシングの何を面白いと感じるか)を再現し、それを AI に試行させると面白いゲームが生まれる、という方法論の検証台。
- 試行錯誤の末、ユーザーが行き着いた仮説: **FSMで人間の面白さを仕組み化することが「面白いゲーム作り」の根幹**。
- ここで成立すれば、同じ方法論を他のプロジェクト(小説・VR・モデリング・画像生成等)に転用する基礎になる。詳しくは [[user-creative-ai-vision]]。

なので「ボクシングシムとして機能を網羅する」ではなく、「FSM がどれだけ人間のボクシングの面白さを捉えているか」が評価軸。FSM 設計を磨くこと自体が成果。

LOG.md からも一貫して読み取れる: ステートマシン図(boxer_fsm.html)・分類表・状態カード等で同じ FSM を 3 通りで表現して整合性を取り、taxonomy で遷移のきっかけを4分類で整理し…と、FSM そのものを大事に育てている。

私の作業姿勢への含意は [[feedback-prioritize-fsm-quality]] に。
