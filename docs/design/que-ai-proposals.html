戦略AI（que）の仕組み — 3つの設計案

「先を読んで手順を組む」ボクサーAIを、どう機械で作るか。あなたの『空振りさせて反撃』の例を1本通して説明します。

設計検討用コードは未変更2026-05-29

いまの壁：今のAIは「今の状態 → 1手」を選ぶ反応型。情報が見えている反応型どうしは、にらみ合い・鏡写しループ・時間切れに落ちる。スタミナや予測を足しても動かなかったのは、足す場所（反応型の決定核）こそが壁だから。

解決の方向：反応層の上に「型(que)の層」を乗せる。＝目的を持って手順を仕込み、途中で相手を観て、分岐し、外れたら捨てて反応層に戻る。これは格ゲーAIですでに定石化された作り方（あなたの誘い→反撃は shimmy、仕組みは ビヘイビアツリー＋相手の癖テーブル）。ゼロからではなく、今ある resQueue を育てる話です。

共通の土台（3案すべて共通）

① 2つの層 ＋ 既存の技FSM

型層（戦略 / que）目的を持つ・手順を仕込む・途中で観る・分岐・撤退
反応層（既定）脅威にガード・相手の空振りに差し込み・間合い維持
技FSM（土台・既存）始動 → 発生 → 硬直

型層が動いていない時は反応層だけが働く＝今の状態と同じ。型層は「これは仕掛けられる」と判断した時だけ前に出る。だから安全側に倒れる。

② 「型」とは何か（あなたの例 ＝ 空振り誘発→反撃）

発動条件：相手の攻撃が届かない・でも遠すぎない間合い

「途中で相手を観て、分岐／撤退する関門」を持つ＝ただのコンボでもただの反応でもない部分。ここが core。

3案の違いは 「関門での"読み"の深さ」と「型をどう選ぶか」。土台（2層・型の形）は共通です。

3つの仕組み案

案1：型を「割り込めるシーケンス」で持つ（最小）推奨：第一歩

既存の resQueue / Playbook / reservationSafe を素直に育てて、上の「型」を実現する。

あなたの例では：関門は「相手は攻撃した？」のシンプルな判定。乗ったら決め打ちの反撃（例：相手の硬直に最速の一発）、乗らなければ撤退。相手が"何を"出したかの細かい読みまではしない。

長所

  • 改造が最小（既存部品の延長）
  • 中身が完全に見える＝人が書ける
  • すぐ動かして観察できる
  • 「誘い・撤退・差し込み」はこれだけで立つ

短所

  • 「読み」が浅い（何の技か細かく見ない）
  • 相手の癖を覚えない（毎回ゼロから）
  • 深い読み合い（裏の裏）は出にくい
複雑さ ★☆☆元ネタ：Behavior Tree（reactive sequence ＋ 条件付き abort）

流用：resQueue=仕込んだ手順 / Playbook=型選び（間合い×有利） / reservationSafe=撤退 / resolvePlan={seq}展開

案2：案1 ＋「相手の癖テーブル」で読み合い推奨：次の層

案1に、間合い別に「相手が何を出しがちか」を記録する表を足し、"読んで刺す"を入れる。

あなたの例では：関門で「この間合いでこいつは何を出しがち？」を表で見て、来た技に刺さる反撃を選ぶ。同じ手が通じなくなってきたら型を切り替え（conditioning）。自分の手は少し混ぜて読まれないようにする。
◆関門：攻撃が来た
↓
相手の癖テーブルを引く（間合い別）→ 来た技を特定
↓
その技に一番刺さる反撃を選ぶ ✅

長所

  • 本物の読み合いの深みが出る（裏の裏）
  • キャラの個性が表で作れる（攻め型/慎重型…）
  • "にらみ合い"を壊す（相手の偏りを突く）
  • 表なので中身が見える（人が初期値を書ける）

短所

  • 部品が増える（表の更新・混ぜる乱数）
  • 調整が要る（表のキーの粗さ＝間合いの刻み等）
  • 観測が貯まるまでは初期値(prior)頼み
複雑さ ★★☆元ネタ：Killer Instinct "Shadow" / 研究AI FightingICE（状況別テーブル＋少し混ぜる＋試合中に適応）

流用：案1 一式 ＋ 新規 opponentModel（間合い→相手の手の頻度表）。{pick} が "自分を混ぜる" 仕掛けに転用できる。

案3：目的から逆算するプランナ後段向け

各型に「前提／効果」のラベルを付け、目的から手順を自動で組み立てる。

あなたの例では：「大技を当てる」を目的に置く → 前提「フレーム有利が要る」 → 前提「相手を空振りさせる必要」 → …と逆算して型を連結。途中で状況が崩れたら組み直す。

長所

  • 拡張に最強（型が増えるほど効く）
  • 型が自己説明的（前提/効果が書いてある）
  • 目的が明示的（意図が読める）

短所

  • 今のサイズ（1次元・型少数）には重い
  • 生成された手順は案1より挙動が見えにくい
  • 「相手の反応」を前提に表すのが少し不自然
複雑さ ★★★元ネタ：GOAP（F.E.A.R.）/ HTN（Killzone 等）

注：前提/効果の"ラベル"だけは案1にも早めに足すと安価で型が自己説明的になる（プランナ本体は後回しでよい）。

推奨の進め方

案1：芯を作る
2層＋型の「仕込む→観る→分岐→撤退」。誘い・撤退・差し込みが立つ。
案2：読み合いを重ねる
相手の癖テーブル＋混ぜる。ここで駆け引きが立ち上がる。
案3：規模が増えたら
型・機能が増えて手で並べるのが辛くなったら逆算プランナへ。

いきなり全部ではなく、案1で芯を作り、観察してから案2。これは「機械的に積んで、結果として面白さが立ち上がる」というこのプロジェクトの方針どおりの進め方です。

補足：避けるもの ＆ 設計テスト

避ける：ゲーム木探索（MCTS等）

研究では最強。だが「中身が見えない・人が書けない・人間離れした最適行動になる」ので、FSMで仕組み化する理念に反する。使うならオフラインの調整役（型の強さの検算）だけ。ライブの頭脳には使わない。

設計テスト：三すくみ ＆ Yomi

三すくみ＝攻め→崩し→守り→攻め…が循環で勝てる関係。これが無いと「最強の1手」が決まり、読みが無意味＝退屈な均衡に逆戻り。

ボクシングなら大まかに：踏み込み攻撃↔誘い/ステップ↔ガード/差し返し。
Yomi＝読みの層（読む→読まれる→読み返す）が深くなる土壌を残す。
