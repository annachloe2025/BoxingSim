ボクサーFSM：4グループ構成図

①ベース状態 ／ ②自発行動 ／ ③被弾リアクション ／ ④強制状態 の4つで構成。被攻撃・ラウンド終了は“どこからでも飛ぶ割り込み”としてグループ単位の矢印でまとめています。

ホイールで拡大縮小／ドラッグで移動 
stateDiagram-v2
    direction LR

    [*] --> Base

    state "① ベース状態" as Base {
        state "待機・構え" as Idle
        [*] --> Idle
    }

    state "② 自発行動" as Voluntary {
        state "前進" as StepF
        state "後退" as StepB
        state "ガード" as Guard
        state "攻撃：始動" as AtkS
        state "攻撃：発生" as AtkA
        state "攻撃：硬直" as AtkR
        AtkS --> AtkA : 始動F経過
        AtkA --> AtkR : 発生F経過
    }

    state "③ 被弾リアクション" as Reaction {
        state "ガード硬直" as GuardStun
        state "のけぞり（通常）" as Hit
        state "のけぞり（カウンター）" as Counter
        state "ダウン" as Down
        state "立ち上がり" as GetUp
        Hit --> Down : ダメージ超過
        Counter --> Down : ダメージ超過
        Down --> GetUp : ダウンカウント
    }

    state "④ 強制状態" as Forced {
        state "インターバル / コーナー" as Interval
    }

    Idle --> Voluntary : 自発入力（移動/攻撃/ガード）
    Voluntary --> Idle : 行動完了（フレーム経過）

    Idle --> Reaction : 被攻撃
    Voluntary --> Reaction : 被攻撃（文脈で結果が決まる）
    Reaction --> Idle : 復帰（のけぞり/立ち上がり完了）
    GuardStun --> Guard : ガード硬直F経過

    Voluntary --> Forced : ラウンド終了
    Reaction --> Forced : ラウンド終了
    Idle --> Forced : ラウンド終了
    Forced --> Idle : ラウンド再開

    Down --> [*] : テンカウント（KO）
① ベース状態 ② 自発行動（自分の意思） ③ 被弾リアクション（相手起因） ④ 強制状態（審判・ルール）

・「被攻撃」と「ラウンド終了」は本来どの状態からでも起きますが、矢印が増えて見にくくなるのでグループ単位の割り込みとして1本にまとめています。
・「被攻撃」を受けた結果（防御＝ガード硬直／通常のけぞり／カウンター）は、受けた瞬間の自分の状態で決まります（詳細は分類HTMLを参照）。
・「ガード硬直」は防御成功の結果なので③に置きつつ、戻り先は②のガードです。
