フェイントを実装する前に、誘発対象となる「能動的な待ち状態」が FSM に存在することが前提。

**Why:**
ユーザーの設計意図は「フェイントは相手がカウンター待ち・読み構えなど能動的な待ち状態にあるときに、その判断を誤発火させて隙を作る」というもの。現状の FSM ではカウンターは被弾時に自動判定されるだけで、能動的な「カウンター予約」状態が存在しない。誘発対象が無い場所にフェイントを足しても、ただの「当たらない攻撃」=損な選択肢にしかならない。

**How to apply:**
- 段階2 (フェイント+頭/腹) の話が出たら、フェイント部分は「待ち状態の設計とセット」として後段に分離することを提案する。
- 「能動的な待ち状態」候補: カウンター予約 (相手の攻撃を見たら出す約束), 上ガード/下ガード待機 (打ち分けに対する反応構え), 反応開始ラグの「次に何が来てもこれで返す」予告状態。
- 段階3 (知覚反応) で「反応に時間がかかる」が入ったあと、それと組ませてフェイントを設計するのが自然。
