# レポート音声化パイプライン 設計書

作成: 2026-05-28

## 目的

シミュレーションが定期的に生成する HTML レポートは内容が密で読むのが大変。
そこで、レポートごとに「耳で聞いてわかる日本語の説明」を自動生成し、
edge-tts（ja-JP-Nanami）で音声化して GitHub Pages で公開する。
外出先（リモート作業時）にスマホで進捗を耳から把握できるようにする。

これまで手動で「ChatGPT に説明を頼んで音声で聞く」をやっていたのを置き換える。

## 確定した要件

- **範囲**: 全レポート。新しいレポートができたら即・自動で音声化＋公開。
- **説明の詳しさ**: しっかり詳しく（長さの制限なし）。聞いて中身がほぼわかるレベル。
- **音声**: edge-tts `ja-JP-NanamiNeural`（インストール済み: `python -m edge_tts` v7.2.8）。
- **保持件数**: 音声は最新10件のみ。11件目以降は古いものから自動削除（古いものは聞かない前提）。
- **スマホでの入口**: 専用の音声ページ `audio.html`（最新10件を再生ボタン付き・新しい順）。ダッシュボード(README.html)からもリンク。
- **トリガー**: フックで完全自動（`Stop` フック）。

## アーキテクチャ

LLM が必要な「説明文の執筆」はセッション内のエージェントが担い、
決まりきった「音声化・ページ生成・整理・公開」はスクリプトが担う、という分担。

### 役割分担

| 担当 | 仕事 |
|------|------|
| **reporter エージェント**（文章担当） | レポートHTML作成時に、聞いてわかる日本語の読み上げ原稿も書く |
| **`tools/publish-audio.js`**（自動処理） | 原稿→MP3化、音声ページ再生成、10件超の削除、GitHubへpush |
| **`Stop` フック** | ターン終了時に `publish-audio.js` を起動（冪等：新規原稿が無ければ何もしない） |

### データフロー

```
reporter がレポート作成
  ├─ reports/日付-トピック.html        （従来どおり）
  └─ audio/scripts/日付-トピック.txt    （新規：読み上げ原稿）
        ↓ ターン終了で Stop フック発火
publish-audio.js
  1. mp3 が無い原稿を edge-tts(Nanami) で音声化 → audio/日付-トピック.mp3
  2. audio.html を再生成（最新10件、新しい順、<audio> プレーヤー、レポートへのリンク）
  3. 11件目以降の古い mp3 と原稿を削除
  4. git add -A → commit → push
        ↓
スマホで …/BoxingSim/audio.html を開く → 上から再生
```

## ファイル構成（新規）

- `audio/日付-トピック.mp3` … 音声本体
- `audio/scripts/日付-トピック.txt` … 読み上げ原稿（文字起こしも兼ねる）
- `audio.html` … スマホ向け再生ページ（最新10件、`publish-audio.js` が生成）
- `tools/publish-audio.js` … 自動処理スクリプト

## 読み上げ原稿のルール（reporter への追加指示）

耳で聞いて理解できる話し言葉にする。具体的には:
- 記号・コード・URL・ファイルパスを読み上げない（数値は言葉にする）
- 段落ごとに区切って自然に話す。見出し記号（#, *）を入れない
- 「何を試して」「何がわかったか」「次どうするか」を、しっかり詳しく
- 宣伝調にしない（プロジェクト理念どおり、観察された事実とその構造的解釈）

## publish-audio.js の仕様

- 実行: `node tools/publish-audio.js`(プロジェクトルートからの相対は `__dirname/..` で解決)
- edge-tts 呼び出し: `python -m edge_tts --voice ja-JP-NanamiNeural --file <原稿> --write-media <mp3>`
- 冪等: mp3 が既にある原稿はスキップ。最後に `git status` を見て、変更が無ければ commit/push しない（レポートやREADME等の変更があれば音声と一緒に公開される）。
- 並び順・件数: ファイル名先頭の日付で降順、上位10件を残す。
- audio.html のタイトル: 対応する `reports/日付-トピック.html` の `<title>` を流用。対応レポートが無ければファイル名。
- 失敗時はメッセージを出して `exit 0`（セッションを止めない）。

## フック設定（ユーザーが手で1回追加）

安全装置により Claude は settings.local.json を書き換えられないため、
ユーザーが以下を `.claude/settings.local.json` に追加する。

```json
"hooks": {
  "Stop": [
    { "hooks": [ { "type": "command", "command": "node \"${CLAUDE_PROJECT_DIR}\\tools\\publish-audio.js\"", "timeout": 120 } ] }
  ]
}
```

（timeout は音声生成に時間がかかるため 120 秒）

## 保留・別管理

- レポートHTML自体は全部残す（ダッシュボードの記録）。10件制限は音声と原稿のみ。
- git 履歴には削除済み mp3 も残るが、1本2〜4MB・個人利用のため容量問題は当面なし。
- 既存の `tools/publish.js`（一般的な手動 push 用）はそのまま残す。
